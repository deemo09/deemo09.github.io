001 使用Github、Hexo部署个人博客

发表于 2024-12-14 更新于 2024-12-24

一、博客文件准备

下载note.js

下载hexo

下载git

以下代码均在git bash中运行

1.新建文件夹hexo并进入，初始化博客，下载主题，该主题详细说明可参照https://xuthus5.github.io/hexo-theme-cactus/hexo/cactus-chinese-support/

2.博客创建完毕，可启动服务在本地浏览器查看http://localhost:4000/

3.生成public文件夹，用于上传至Github产生静态网页

二、新建github仓库

1.新建github仓库，名称为username.github.io，勾选产生readme

2.新建一个分支，用于产生网页，主分支main用于同步和修改

3.在项目Setting-Pages将Branch设置为新建的分支

三、将本地内容上传Github仓库

1.配置git，注意使用Gthub提供的私密邮箱防止暴露隐私

将产生的id_rsa.pub内容在Github网页里填入到Github账户SSH设置中

2.新建同步文件夹sync并进入，用于存放仓库文件

可以看到当前目录下存在./ ../ .git/

3.链接项目，重命名分支，将云端项目拉取到本地

4.将hexo g产生的public文件夹内容复制到当前文件夹，随后将这些内容同步到云端仓库main分支

5.如果使用的是个人邮箱，会提醒暴露了私有邮箱自动拦截，需要更换Github提供的隐私邮箱重新push

成功push到仓库main分支

6.将main分支的改动同步到网页分支，在仓库网页中选中Pull requests，新建一个pull，base选网页分支，compare选main，然后提交同步，一直点确定，成功同步

7.对个人博客hexo文件夹修改后，使用clean然后重新generate，把新生成的文件复制到sync文件夹，并重复上述同步过程
